List of standard modules and models

Here, we list all the modules and model types, that are shipped with the AgentLib core. The identifier specifies the string that should be used in type to use the module or model (see example at the Reference bottom of this page)

Example

When using modules, they are identified by the "type" keyword in the configuration, which should include an identifier from this table, or a custom injection (combination of file and class name). For example, to run an agent with a simulator and an FMU model, the config should look like this:

{
  "id": "myAgent",
  "modules": [
    {
      "type": "simulator",
      "model": {
        "type": "fmu",
        "path": "path/toFMU.fmu"
      },
      
      "...": "remaining simulator config goes here"
    }
  }
